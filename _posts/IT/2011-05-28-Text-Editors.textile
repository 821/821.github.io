文本編輯器是神馬？可能菜鳥不曾注意這問題，因爲在 Windows 和 MAC 下大家都會一般習慣性的裝上一個 Office ，中國的電腦商家和修理工甚至更直接，幫你裝一個帶 Office 的系統完事，很多號稱純淨版的 Ghost 系統都會附上自選安裝 M$ Office 。但是 Office 問題很多，比如性能差，佔用資源多，而且穩定性也實在不怎麼樣。另外雖然 Office 具有排版的功能，卻沒有代碼高亮，不適合進行編程、網頁編輯等等處理，這時候就需要一款專業的文本編輯器來輔助了。
我老人家使用各種文本編輯器的經驗不算豐富，但也不比全世界九成九的人少。我老人家從不進行複雜編程，頂多寫寫論文、網頁代碼、小腳本的人，那這篇文章所期待的讀者亦是如此了。文本編輯器、輸入法、鍵盤、鼠標，是數字化時代的筆，找到適合自己的文本編輯器很重要，而且體驗可以說無法複製，每個人都有自己的手感和看法。本文更多的是羅列各款文本編輯器在中文語境下的特色優缺點，並給出自己的選擇。

h2. 起式

很多拳法都有起式，讀者若看過太極拳的視頻，那前面更是要有一段比較長的時間給拳師醞釀。問自己一個問題：我需要什麼？
我主要用來碼字撰文、寫小腳本、寫小型網站、修改輸入法詞庫和碼表，需求如下：

01. 舒適的學習曲線，權重 6
02. 高性能——啟動快，秒開 3MB 以下文件，大替換和搜索速度理想，權重 5
03. 代碼及字符高亮、代碼折疊等——簡單編程和撰文所需，我喜歡高亮標點符號，權重 8
04. 字體、顏色等可視效果——關乎長時間工作時眼睛疲勞問題，最好能方便的編輯主題，權重 10
05. 自動完成、代碼縮寫——看視頻很美好，其實不編程的不常用，權重 1
06. 中文支持——多種編碼的中文顯示、漢字渲染效果、中英混雜文章的折行，權重 65535
07. 簡單列模式——能列刪除就好，權重 3
08. 多步撤銷——這功能都做不出來，不要做編輯器了，權重 10
09. 美好的插件管理機制——免重啓使用、插件控制面板，權重 1
10. 搜索與替換——正則表達式、搜索高亮、大批字符串對替換（詳後），權重 10
11. 舒適的操作——鼠標、工具欄、 Windows 式快捷鍵、組合式快捷鍵，權重 10
12. 便捷打開文件——最好是收藏夾+快速打開命令，開命令要簡潔且支持模糊搜索，權重 6
13. 自動保存——雖然我有手動保存強迫症，不等自動保存就會手動保存掉，但既然有強迫症，權重 8 好了
14. 宏，權重 1
15. 開源免費，權重 3
16. Windows 版，權重 65535
17. Linux 版，權重 2
18. 保存文件開啓狀態，權重 5

以下是我不太關心，但很多人關注的：

01. 界面
02. MAC 版本
03. IDE
04. 網頁預覽和代碼編譯——我衹需要能夠調用其他工具，軟件本身不必集成
05. 編輯區最大化——很多人喜歡殺掉工具欄和菜單欄，我沒有這種愛好
06. 統計——我使用專業的統計工具，不過狀態欄必須顯示光標在第幾列
07. 書籤
08. FTP
09. 多行編輯——不編程，所以不需要
10. 函數跳轉——同上
11. 多重剪貼板
12. 十六進制


h2. 黑虎掏心 Notepad

哦哦哦，你見過比他還爛的編輯器嗎？？？反正我沒見過。
按理說犧牲了功能的，性能應該要好。可是他偏偏就不行，隨便一篇網上下載的中長篇小說，拿下來想替換幾個字，他就給你裝死……
可是，如果你有虛竹的內力，一招黑虎掏心也能跟鳩摩智打。
好吧，我不是高手，需要高級招式。而且就算是虛竹，不學生死符大概不好收拾丁春秋吧，聽說最近人家還學了降龍十八掌哦。
或許我們都錯怪了 Notepad ，他衹是一個能夠快速啓動，然後編輯迷你文件的小工具，拿來看長篇文章是對他的褻瀆。

h2. 乾坤大挪移 Notepad2

Notepad2 和諸般強大的文本編輯器比，顯得非常不給力，但比起 Notepad ，又好了太多，而且還是單文件的。於是我把他下載下來，改名，替換掉 Notepad 。哦也，從此電腦上的 Notepad 變成了一個有列模式、可高亮代碼、無限撤銷的高性能編輯器。不過，功能基本上也就這麼多了……
如果需要更多功能，可以選擇其他，或者用他的 <a href="http://code.google.com/p/notepad2-mod/downloads/list" rel="external">MoD</a> 。

h2. 破劍式 Notepad++

Notepad++ 多次被評爲最佳免費文字編輯器，運用廣泛。他速度快，界面清爽，不配置也很好用，是極好的入門編輯器。
Notepad++ 在代碼折疊方面做得很好。不但能折疊，而且默認情況下對一些語法還有非常漂亮的配對高亮。這對編寫複雜結構的網頁頗爲有用，有時 div 多了，改改就亂了，如果一點就看到另一頭，就能比較輕鬆的看到另一頭的位置，縮排規範的話，立刻會發現問題。
我也非常喜歡他的選中高亮，很多軟件選中一個詞之後，只有被選中的詞高亮起來，而 Notepad++ 是高亮所有同一詞彙。
問題來了：編輯中英混雜的文章時，會把連續沒空格的中文視爲一個單詞，自動換行非常囧。諷刺的是，作者是臺灣人。我真的想問一問：你自己到底用沒用過 Notepad++ ？

h2. 降龍廿八掌 UltraEdit

曾幾何時， UltraEdit 在業界地位極高，大有圖形化設置的編輯器中老大的地位，基本上新版一出，盜版跟進。原因有幾點：出道早，易上手，支持十六進制，列模式，自定義配色，當時還真沒幾樣編輯器能做到。
悲劇的是，長串中文在 UltraEdit 還是一個詞。他的默認工具欄很爛，保存都沒有，就一堆不知所謂的爛圖標，不知作者在想什麼……他的 html 工具欄還行，不過我用的機會不多。現在 UltraEdit 最大的優勢也就是重要功能他都有，外加一個極品列模式了。軟件可以說已經被作者搞壞了，整個界面凌亂不堪，相同功能不同圖標的情況比比皆是，看着就眼脹。個人認爲他應該針對非 html 大膽砍掉一些累贅的東西，回歸正常的工具欄、狀態欄。

h2. 八荒六合唯我獨尊功 EmEditor

泥轟人作品，可能是亞洲語系長期被邊緣的緣故吧，作者對各種語系都做得相當週到，中文的折行和渲染在這裏得到了完美解決。軟件非常細膩，而且設置方面都提供了 GUI 設置的口。我個人很喜歡他的工具欄可以和菜單欄合併成一行這一特色，體現了 GUI 的高級境界。同時，其強大的右鍵菜單也體現了鼠標流的最高境界。早期的 EmEditor 由於軟件機制問題，需要設置纔能更高效的使用，但現在安裝包集成了插件，那些原本埋藏甚深的功能已經相當明顯了。
EmEditor 的替換也代表着當代單文件替換技術的最高水準。可能有的人會聽過 Search & Replace 和 Actual Search & Replace 這對機油，號稱批量查找替換界的倚天劍和屠龍刀。不過我把他們都刪掉了，因爲我很少有機會去替換大批文本文件，但我經常替換一個文本文件的多對字符串。一個合格的文本編輯器應該具備正則表達式替換，而一個優秀的文本編輯器似乎就不應滿足於正則表達式，而應該引入批量字符串對的替換了。比如，我想把全拼碼表換成雙拼，那就有三四十對字符串要替換，一對對輸入、替換當然可以，但實在太麻煩了。而且我可能有很多個碼表要做替換工作，那麼保存替換腳本也是很重要的。也許有人會舉起小手提到馬健寫的 textforever ，不過那玩意的替換跟 Search & Replace 之類差不多，還不支持非系統默認編碼的文件，而且替換腳本的格式不懂是十六進制還是神馬，反正很難編輯，只能一對一對添加。 EmEditor 就有字符串批量替換功能，替換腳本是 tsv 格式，非常方便修改——就憑這個功能，我是不會刪掉我硬盤上的 EmEditor 的。當然了，宏也是各家文本編輯器實現此功能的手段之一，可是到底不如原生支持來得方便，還要自己寫。 EmEditor 還讓我比較感動的是他可以找出帶有符合字段的行，打上書籤，然後刪掉書籤所在行或者把書籤所在行全部複製到一個新文件，給我整理詞庫帶來不少便利。
大綱 (Outline) 功能也很好用，可以通過自定義設置，把需要凸顯的章節露出來，只要點右側的大綱欄，就能快速跳到你設置的點。這不同於書籤功能。書籤是你把光標移動到某處，然後設置。大綱則是在配置裏設定好關鍵字符組合，以後每次出現該字符組合，不用再設置，自動大綱化。
代碼片段 (Snippets) 也是 EmEditor 的特色功能（好像原始發明人是 TextMate ，但那貨是蘋果下的軟件，跟我沒關係），有了此物，你甚至覺得 Zen Coding 的 html 簡寫工具都是多餘的，因爲論可定制性，還是代碼片段來得美妙。但我不知道爲什麼作者把這功能做成插件，讓我每次想用都要啟動插件，而且啟動插件會出現一個側邊欄……
EmEditor 功能上最大的缺陷是大概就是沒有收藏夾罷，收藏夾可以用項目 (Project) 插件來代替，不過還是真正的收藏夾更好用。另外，做大型項目的程序猿也會說他不適合打造成 IDE 。
EmEditor 是一款近乎完美的大眾化文本編輯器，是所有文本編輯器中最積極吸收他者優點的一款。儘管其存在部分情況下 CPU 佔用莫名飆高不下（之前我的電腦比較菜，雙核，主頻不到 2GHz ，內存 2GB 還被 APU 喫掉一些，就有問題；現在換到 G1620+H61+8GB RAM 的平臺，就沒問題了）、冷啓動較慢（原因是文件較多，而我又沒有固態硬盤）、與某些輸入法不太合作、沒有收藏夾功能、 Snippets 功能實現方式詭異等缺點，他仍然是最適合編輯長篇文章，並且兼顧編輯代碼的工具。

h2. 神行百變 EditPlus

EditPlus 下載回來，我一開，哇靠，山寨版 EmEditor （作者是棒子，果然是山寨 EmEditor ）。
EditPlus 小巧輕便性能高，界面和功能與 EmEditor 頗相似，卻更加容易上手，門檻低，速度快，韋小寶都能用。他的 html 工具條做得頗爲細膩，還有個「素材」功能，感覺有點像 cheatsheet 。軟件與 FTP, Git, SVN 有緊密結合，爲開發者提供便利。
EditPlus 有一個其他軟件沒見過的功能——窗口平鋪。比如你開了十個文件，只要輕輕點一下平鋪，他立刻把這十個窗口根據一個比較合理的配置全部呈現在用戶眼前，有點 MAC 的感覺是不是，還可以自定義每個窗口的大小。快捷鍵也是他的特色，你可以設置菜單欄裏所有命令的快捷鍵，只要你記得。如果寫有函數的腳本，他也支持函數定位。
他的「工程」比 EmEditor 的更細緻化，還整合進菜單裏。添加文件夾到工程裏後，點擊該文件夾，軟件會直接把整個文件夾的文件（不包括子文件夾）全部打開，無比凌亂。不過如果衹添加常用文件到工程，這個工程還是可以當作一個不太好用的收藏夾。
EditPlus 的可定制性稍差，一些方面不如 EmEditor 細膩，代碼高亮很原始，主題基本上是免費軟件的水平，嗯，缺點內功……總體來說， EditPlus 可以看成是 EmEditor 的 html 加強但全局劣化版，適合前端開發和文本瀏覽。

h2. 降龍十八缺三掌 PsPad

被譽爲 UltraEdit 的最佳免費替代品，到底有沒有這麼厲害呢？
我嘗試之後，覺得他跟 UltraEdit 的差距還是比較大的，很多功能都沒做好，甚至自動換行都曾經被作者刪掉，簡單的代碼折疊居然都沒有。而且不是 unicode 軟件。結果 PsPad 就像缺了幾招的降龍十八掌一樣，怎麼看也就是黎生的水準（知道黎生是誰嗎，補習一下射雕吧）。

h2. 北冥神功 AkelPad

新興的免費瀏覽器，插件多，速度快。用這軟件嚴重推薦烈火漢化版，總是加上很多插件（烈火作品吸引我的從來不是漢化本身，而是破解和加強）。如果作者有心把插件整合一下，再把設置和功能調用做友好些，完全可以賣個好價。剛接觸 AkelPad 、第一次寫本文的時候，他還顯得頗簡陋，衹覺得他很有發展前景，關注了一下。但他本身實在進步神速，而且烈火似乎也相當積極把插件加進去，因此時隔一年再用烈火版，他忽然變成 GUI 編輯神器了。
他很早就實現了拆分窗格功能。當我們編輯長代碼或長篇文章的時候，如果想同時瞻前又顧後，拆分一下窗口就可以同時了，最多可以拆分成四個窗口，絕對夠用了。另外他支持最小化到任務列，這是一般文本編輯器所不具備的。
不過最吸引我的還是收藏夾。我有一些經常編輯的軟件，總數還不小，資源管理器又衹管一個文件夾，這實在讓人苦惱。有了收藏夾，問題就解決了。
AkelPad 選項比 EditPlus 還少，但有些更像是不同編輯器的取捨的東西，他也允許設置。比如，你可以設置查找替換是否循環，再有空格符、製表符這些，雖然軟件有內建，都是可以通過插件修改的。最神奇的是他的折行方式選擇，選按符號，就是我比較滿意的中文折行，選按單詞，就是類似 UE 等中文支持較差的軟件的方式。開發者沒能像 EmEditor 和 EditPlus 那樣完美解決中英混雜文章的折行問題，卻又提供了另一思路。
不過，目前 AkelPad 還有一些不足之處。比如他的列模式就不如 UltraEdit, EmEditor 強大。代碼高亮和主題的可定制性也弱於成熟軟件。他的代碼編輯體驗還算及格，我比較喜歡他的代碼大綱，用起來還挺像 EmEditor 的大綱，也提供了輸出爲富文本的功能。如果作爲輕量級代碼編輯器和寫作工具，他都是可以勝任的。

h2. 葵花寶典 VIM/gVIM

VIM 強大，又不是那麼好學，門外漢往往敲了半天都輸不進一個字。練對了就東方不敗，沒練對就林震南。不過平心而論， Windows 下的 gVIM 不是很快，特別是編輯大文件時，體驗明顯不如 EmEditor （當然這廝的實現手段很偷懶）。
VIM 的特點在於命令操作，使用 VIM 可說是一種鍵盤操作體驗——前提是記得命令。由於 VIM 把操作分爲三種模式，所以不太用組合鍵，手指的壓力會減小很多。我用得最熟的命令就是 @dd@ （比 UltraEdit 的 @Ctrl+E@ 好按，更不必說 EditPlus 的 @Alt+Shift+Delete@ 了，誰想出這麼二逼文藝的組合鍵啊，而且 @dd@ 可以用來多行刪除）。還有比如想截取一個比較大的文件的若干行，那必是 VIM 最給力，速度快不說，幾個命令也不難記。但比如替換之類的，我就覺得非常不給力。再有模式切換也有討厭的地方，很多重要功能需要冒號進入模式，這一點非常不好，因爲冒號的輸入需要按著 @Shift@ ……
鑒於 VIM 的命令較好用，很多現代化文本編輯器都會集成 VIM 模式。有的做個插件，有的原生支持。所以其實不用他也不必羨慕。好按的快捷鍵已不是 VIM 獨有。
由於 VIM 是兩大最古老的編輯器之一，插件極多，高度可定製，因而非常強大。遺憾的是有些問題還是無法在上面解決。比如他的屏幕滾動極突兀，還有高亮當前行會高亮折行後的整行，非常不適合喜歡寫長段落的本人。他的模式化單鍵操作固然有其優越性，然而脫離了組合鍵，中文輸入法就會和他衝突，因而需要不斷的切換輸入法的中英模式，這也是非常討厭的。故而，我個人認爲 VIM 是極度不適於碼字的，他是程序猿和短段落西方文字撰寫人（比如西方詩人）的軟件。
VIM 高手一般會把工具欄都殺光，欲練神功，黃且暴力，所以如果還想跟鼠標洞房花燭，那就衹能找楊蓮亭了 <img src='{{ site.url }}/assets/Smilies/o042.gif'> 。

h2. 太玄經 Emacs

偽裝成編輯器的操作系統，如果你會設置，他簡直無所不能。
我注意到 Emacs 是因爲 TeX 。寫長文章修改的時候比較怕遇到代碼密集區，簡直不堪入目，所以最好能有一款一邊寫一邊 preview 的。我看了一下，基本上都是收費還找不到破解版的，或者就是問題比較多的，唯獨 Emacs 配合一些插件貌似沒人非議，所以很想試試看。裝上之後，我就知道不可能了。 VIM 已經夠麻煩，沒想到 Emacs 更麻煩，上來就要進行一大堆的設置，設置就要 Google ，很浪費時間。你就不能發一個編輯好參數文件的版本麼，高手愛怎麼設置是他的事，可是你總該給一個新手能用的吧。另外，同是鍵盤操作， Emacs 不如 VIM 好用，除非裝上腳踏板，特別對我是我這種手指很短的人（研究表明，手指短的人某個器官也短，很慶幸我不是男的，不然多難爲情啊），最討厭按 @Ctrl@ 了。如果說 VIM 是二逼青年的編輯器，那 Emacs 絕對是二逼文藝青年的編輯器。
Emacs 還有一個讓我無法忍受的地方，那就是不穩定感。不知道是本身太慢還是怎樣，衹要我用來操作大概 1MB 的文件，就非常容易卡住，然後 CPU 佔用 50 左右，衹能強制關掉。當然這是 Windows 下的情形，我不確定在 *nix 下如何。
當然了，如果讓我免學習代價完全掌握一款文本編輯器，我當然會選他。問題是他的時間成本眞的太高了。學會了 VIM 或者 Emacs 的人一般不會換成別的編輯器，原因是這兩款的學習成本太高，學會了還換那也太不拿時間當生命了。 Emacs 比較擅長和 *nix 系統本身進行深度整合，在 Windows 下處於被閹割狀態，所以不建議在 Windows 下使用——或者說， Windows 版的 Emacs 僅僅是讓其用戶偶爾在 Windows 下工作時沒那麼無所適從的一個補充品。

h2. 岱宗如何 SciTE

跨平臺編輯器，說起來他和 Notepad++ 及 Notepad2 還是兄弟，而且是嫡子——他們老爸叫 Scintilla 。但和 Notepad++ 定位不同，他注重定制，需要到參數文件自行配置，以及用 lua 進行功能擴展。但 SciTE 的配置文件列出了幾乎所有可定制內容，並將沒用到的都注釋起來，所以用戶可以非常輕易的在這模版下摸索，而 VIM 和 Emacs 都沒有這麼做（也不好做）。故此， SciTE 的地位非常尷尬，功能不到極致，友好度又不及其他，所以強大是強大，小巧是小巧，免費是免費，卻無法取得統治地位。
由於開源且 lua 插件比較容易做，在很多語言及腳本圈子內部， SciTE 是很受歡迎的，還有人提供配置好了的下載，可見他對代碼編輯的優勢。但這些修改往往不觸及內核，以致他的修改版都顯得雷同且有些僵化。插件作者似乎比較熱衷於對自己常用的某款語言進行優化，而一般性的插件則甚少開發，導致目前功能性質的插件還比較少。另外一個明顯而且嚴重的缺陷是他並不支持跨行的正則表達式搜索（ Notepad2 亦不支持，但 Notepad++ 支），這可以說是一個我無法忍受的硬傷，所以無法作爲第一編輯器。
我不太喜歡 SciTE 的文件結構，下載下來一解壓，根目錄一大堆語法文件，我每次都要狂刪一陣。（他的俄羅斯 mod —— SciTE-ru 就杜絕了這個現象。）但是軟件的引擎很好，經過精心設置可以勝任絕大多數編程相關事務，也可以用來瀏覽文本。

h2. 半部葵花寶典+楊蓮亭 Editra

原生支持 VI 模式，可以在一個圖形化的窗口中直接選擇不同語言下的顏色方案，並且實時預覽顏色效果。他還是三大系統通吃的，這種編輯器往往直接編輯配置文件，而他堅持 GUI 。
遺憾的是，中英混雜文件的折行問題又出現了……而且似乎對中文顯示有比較多問題。

h2. 沖靈劍法 Sublime Text

忽然火起來的一個編輯器，介紹說，他介面是仿 TextMate 的，原生支持 VI 模式，支持 Emacs 風格的組合式快捷鍵，快速定位函數，多行編輯，快速打開文件，還能右鍵打開鏈接裏的本地文件，而且雖然使用文本來進行設置，不過註釋很詳細，不會給用戶帶來困擾。於是我下載來看了看，結果發現槽點實在太多了，多到我很困惑，何以一個做了那麼多特色功能的軟件，基礎功能竟然這麼糟糕？
首先他的中文支持是最差的，不但中文折行渣，而且不能顯示 GBK ，衹能用插件轉成 unicode 。單是這理由，我就把他槍斃了。（網上有解決中文折行的方法，但是我放在我的 textile 裏沒成功。問題是我中英混排的文章基本上都是 textile 格式的。）
這貨右側有個文件縮影，截圖看起來很牛，實際用完全垃圾。根本看不清，還好可以關掉。我找了半天也沒發現工具欄，最後百度發現需要用 @Ctrl+Shift+P@ （沒錯， Emacs 的快捷鍵跟他比起來簡直弱爆了）臨時呼叫出一豎條，還一眼望不到底要下拉或敲字那種，權當工具欄。這是我見過最醜、最難用的工具欄。
很多人說他的配色方案很好，但要自定義還要改代碼，很不方便，而且我還沒聽說有什麼編輯器不能下載配色方案的。
還有人談到速度。啓動的表現確實還行，但是開大文件不快，而我對啓動不大關心，因爲我開機就開文本編輯器，關機纔關他，但是文件倒是開開關關比較多。其實我要開的文件也不很大，一般都五兆以內，但我測試覺得他明顯慢了——開大概 500KB 的文件就有比較長的卡頓，甚至有時開很小的文件——比如本文的源文件，也要等上好一陣子。再說文本編輯器有幾個打開特慢的？又不是辦公軟件。也許這是我的系統的緣故，畢竟該軟件是在 MAC 上開發的。
我沒實際使用，衹進行了常用功能的測試，覺得功能欠缺嚴重，免費的都懶得用，何況還要破解。如果編輯純英文的文檔，他還有點用；如果拿來編輯中文文檔，那眞是華而不實了。

h2. 乾坤大挪移 jEdit

VIM 和 Emacs 的粉絲將他們捧上天，但如果這麼好，再加上二者免費、跨平臺、還這麼老，早就千秋萬載一統江湖了，又何來「其他編輯器」之說？不過東方不敗沒做到，無名太監沒做到，君子劍先生甚至死於小菜鳥儀琳之手，小林子被關在西湖底。很多人把不用 VIM 和 Emacs 歸結爲太懶，這衹能說明他們在拒絕理解他們無法理解的事物。人們發明鼠標，就是因爲單用鍵盤不夠方便。很多事可以單手鼠標完成，何必雙手？你以爲人人都住在熱帶，不會遇到冬天不想把手伸出被子的情況嗎？鼠標還能拿到離自己最近的地方，然後把腳搭在舒適的地方（如桌子上），鍵盤流恐怕很難罷？
jEdit 就是一個強大但同時滿足鼠標流的現代化跨平臺編輯器。雖然用戶還較少，插件不多，軟件已經有很多方面做得不錯。他的插件管理非常方便，插件管理器裏勾選下載後，不必重啟軟件即可使用，而且插件的文檔、設置都跟編輯器本身的文檔、設置整合在一起。另外我也喜歡他的收藏夾，這好像是唯一一個自帶收藏夾的編輯器？還有很多特色功能都可以在他這裏找到，比如原生 VIM 模式之類。
但缺點也很多。他需要 Java 環境，冷啟動慢（熱啟動沒問題）。由於亞洲用戶還不多，中文字體渲染也有點問題，有的渲染會導致部分中文字符閃爍，且這些渲染會導致使用中文輸入法編輯時，自動折行出現一些問題，用了渲染後鼠標滾輪滾動瀏覽不流暢。輸入法顯示也不對勁，這會影響作者的打字效率——除非使用重碼率極低、可以盲打的形碼。另外，他有些設置很奇怪，比如行末標記是個很小的點，太不明顯，而且位置跟英文句號很接近，容易搞錯。空格和製表符顯示並不原生可見，插件作者審美有點毛病，用非常不舒服的一個小圈表示空格。還有他不適合編輯過大的文件，開大文件還要設置半天。主題也比較簡陋，這方面很像 AkelPad 。軟件看起來也比較簡陋，期待後續發展。
強大是強大，可這苦逼的娃就像乾坤大挪移，有幾句話你就是身負九陽神功這種萬金油絕學的張無忌也練不成。相信放到性能更好的 *nix 系統下他的表現會好些，不過我現在是 Windows 用戶。

h2. 小無相功 EverEdit

原名 LotusEdit ，在我開始撰寫本文時，他剛改名不久，各方面都不大完善，但後來積極借鑒，越來越強。
軟件作者是中國人，軟件目前免費，但計劃人氣高後轉爲收費軟件。作者自稱反思 EditPlus 而來，但我認爲其功能上主要借鑒對象是 EmEditor ， EmEditor 的主要功能他都有，圖標也很像，而且還抄了少量 EmEditor 沒有實現的功能。他有 Snippets, Project, Outline, 外部工具，批量替換，書籤所在行集體刪除與複製，任意字符高亮，以及可自定義有些類似 Emacs 的組合式快捷鍵，但凡抄了的，相似度 90% 。當然他也確實抄了 EditPlus ，不過主要是抄 UI ，比如他的設置界面和工具欄，就非常相似。在抄了很多功能的基礎上， EverEdit 仍然保持高性能和低佔用，作者的開發熱情很高，積極吸取其他軟件的優勢，所以這款算是 EmEditor 的優良替代品。特別值得一提的是，他的替換是我在 Windows 下試用過的文本編輯器裏最快的一個，我試過處理四十多萬行的大漢和辭典詞條，運用正則表達式的方式一次性做四十多萬個刪除動作，十秒左右就完成了，比 EmEditor 快很多。但是讓我感到頭痛的是內建的刪除重複行功能，竟然會泄漏一些行。還有奇怪的是既然說要仿 EditPlus ，那就應該仿他做得比較好的工程啊，但是最後仿的卻是類似 EmEditor 的那種，然後叫快捷目錄。
EverEdit 在趕新潮方面做得不錯，比如 Markdown ，他就原生的附帶了高亮文件和預覽功能。
EverEdit 還有一點給我印象頗佳，那就是他的編輯區主題設置機制。他支持圖形化配置，也支持編輯主題文件配置，可配置項目充足，文件結構清晰，操作便捷，還能複製粘貼顏色，很適合折騰。在這一點上，我的觀念似乎和作者達成了共識，即，圖形化配置是方便用戶的方式，而文本配置的方便程度有限，則應該是補充。
最後，我要吐槽的是，這貨竟然沒有自動保存這麼基本的功能？另外右鍵無法直接刪除——他竟然把刪除做成一個子菜單，要等他彈出纔能操作。他的工具欄對自定義命令的支持很弱，自定義圖標僅支持放 10 個。文字渲染引擎似乎也存在問題，我寫文章經常用到破折號，在一個打開得比較久的 EverEdit 實例裏，有時輸入破折號會無法正常顯示，需要再打一個字纔正常回來。還有既然是要替代 EmEditor ，他理應支持作爲 Git 的編輯器，但是由於 Git 本身比較討厭的需要加載 /sp ，而很多文本編輯器不支持，不幸的是， EverEdit 也在其中。因此，想要用他完全替代 EmEditor 是不可能的，但可以看作是一個輕量快速而略有先天性殘疾的 EmEditor 。

h2. 全眞劍法 RJ TextEd

又一款開源編輯器。這個編輯器比較針對網頁開發，可以輕鬆的使用 Chrome, FF, IE 來預覽。有一個 html 菜單，功能上和別家的 html 工具欄相仿，但更強大，也更整齊。另外，他還可以輕鬆添加 CSS, JS, PHP 的常用語法。編輯器內置雙窗口文件管理器，比較方便進行文件移動、複製、刪除等操作。
比較有趣的一個功能是統計。一般文本編輯器的統計功能比較簡陋，不包含詞頻，這一款不但有詞頻，而且給出百分比。不過這個功能是基於空格、換行、標點的，所以中文不好使——他的中文支持眞的有待提高，中英混排折行完全不行。
說一下缺點罷，速度較差，主題配置莫名其妙，最奇怪的是他的工具欄竟然沒有鼠標懸停提示，操作上和成熟編輯器也有一定差距。
也許他僅僅是還沒找到他的玉女劍法來雙劍合璧。

h2. 玉女劍法 MadEdit

開發已經停滯了的一款開源跨平臺編輯器，語法高亮文件頗有缺失，但仍有值得關注的地方。
他的速度很快， CJK 支持良好，編碼檢測強大，有着中規中矩的正則表達式替換，主題和高亮簡單但不算太簡陋，也具備列模式和十六進制模式，軟件內部還具有繁簡日互相轉換和全、半角切換功能。
不過他的特色大致也就這麼多了——怎麼樣，是不是和 RJ TextEd 很互補呢？不過如果單獨使用，輕快且 CJK 舒適的 MadEdit 顯然是更優選擇。

h2. 收式

文本編輯器和輸入法都是非常私人的東西，沒有絕對的好，但似乎有絕對的差。對我而言， EmEditor 和 EverEdit 是最舒適的， AkelPad 也不錯。平時以 EverEdit 作爲碼字、寫腳本、寫網頁、簡單文本處理的工具，而 EmEditor 處理一些 EverEdit 處理不好的文件，任意一款輕快的編輯器作爲超短且不需要高亮的文檔的簡單編輯器。
如果使用 Linux ，我會比較願意使用 Emacs 和 jEdit ，另有備胎 Sublime Text 。目前暫時找不到一款讓我完全滿意並停止尋找之路的編輯器。
